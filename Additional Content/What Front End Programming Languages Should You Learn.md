Before we explore the front end languages, libraries, and frameworks to learn in 2025, it is important to understand what they are and the differences between them.
#### Languages 
These are core programming languages that developers use to write code/instructions for the computer to execute, they have ==syntax== and ==semantics==.

Examples include:
- JavaScript
- TypeScript
- Dart
- Kotlin

#### Libraries
These are collection of ==pre-written code== developers use to perform specific tasks
They add extend functionality of a language by providing functions, methods, and classes

- React
- Anime.js
- Chart.js
- jQuery

#### Frontend Frameworks
These provide structured approach to developing applications

- Vue
- Angular
- Flutter

#### Difference between `Language`, `Library` and `Framework`

|Concept|Description|Key Role|
|---|---|---|
|Language|Used to write code and define program logic|Core syntax/tool for building software|
|Library|Provides reusable functions and utilities|Helps developers avoid rewriting common functionality|
|Framework|Offers a structured approach to building applications|Defines architecture/patterns for development|

#### Importance of Frameworks and Libraries
- Frameworks and libraries provide a structured approach to application development
- They include pre-written code and reusable functions for common tasks
- They establish coding standards and architectural patterns
- They save developers time and effort during development
- Many frameworks include built-in security features for safer code execution
- They improve scalability for growing applications
- They help optimize performance
- They contribute to better overall software quality
